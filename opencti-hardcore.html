Qué es OpenCTI

OpenCTI es una plataforma open source para gestionar conocimiento e indicadores de inteligencia de amenazas, construida alrededor de un grafo de conocimiento y de una API GraphQL para operar datos, relaciones y flujos de CTI.

Por qué impone

La propia documentación la plantea como un stack con plataforma central, workers y conectores, apoyado en RabbitMQ, Elasticsearch, Redis y S3/MinIO; es decir, no es un visor, es una arquitectura de ingesta, correlación y producción de inteligencia.

La bestia por dentro

🖥️

Core Platform

La UI y la API GraphQL viven aquí; es el punto central al que hablan conectores y operadores.

⚙️

Workers

Los workers consumen mensajes de RabbitMQ para procesar escrituras asíncronas y escalar el rendimiento de ingesta.

🔌

Connectors

Los conectores importan, enriquecen, exportan o consumen streams desde y hacia la plataforma.

🔎

Elasticsearch

Hace de base principal de búsqueda y persistencia del conocimiento del stack OpenCTI.

🧵

Redis + RabbitMQ

Redis gestiona sesiones y flujos de eventos; RabbitMQ articula la mensajería entre componentes y conectores.

📦

MinIO / S3

MinIO almacena ficheros y artefactos crudos como informes o adjuntos procesados por la plataforma.

SpiderFoot / Shodan / OSINT externo

OpenCTI permite usar conectores de importación y enriquecimiento para traer datos externos y ampliar entidades ya conocidas, incluyendo integraciones de enrichment como Shodan según la tipología oficial de conectores.

Graph + contexto + priorización

La gracia no es almacenar IOC sueltos, sino relacionarlos con malware, campañas, actores, técnicas, informes y observables para contar una historia operativa y accionable.

Casos internos

Podemos usarlo para CTI, incident response, gestión de vulnerabilidades y compartición de inteligencia, que son precisamente casos de uso documentados por OpenCTI.

De hallazgo a decisión

Lo que aparezca expuesto, correlacionado y repetido entre fuentes deja de ser ruido y pasa a convertirse en prioridad real para hunting, hardening o respuesta.

Qué tipo de conectores encajan en la operación

EXTERNAL_IMPORT

Sirven para traer datasets o fuentes remotas y convertirlas a STIX2 dentro de OpenCTI.

INTERNAL_ENRICHMENT

Escuchan nuevas entidades o peticiones del usuario para enriquecer datos existentes con fuentes externas.

INTERNAL_IMPORT_FILE

Procesan ficheros subidos por UI o API, incluyendo STIX, PDF, texto o HTML.

INTERNAL_EXPORT_FILE

Permiten sacar datos hacia formatos útiles para compartir, revisar o intercambiar inteligencia.

STREAM

Consumen eventos en tiempo real para reenviar inteligencia o activar automatizaciones aguas abajo.

XTM Hub

La librería de integraciones de Filigran centraliza el catálogo de conectores disponibles y su uso.

Por qué no cualquier empresa se mete en esto

Porque exige stack

El despliegue recomendado ya implica varias piezas de infraestructura, escalado horizontal posible y dependencias con memoria y disco nada despreciables.

Porque exige disciplina

Sin taxonomía, sin fuentes claras y sin procesos de revisión, el grafo puede llenarse de ruido más rápido de lo que genera inteligencia útil.

Porque exige operación

Los workers, conectores y pipelines viven mejor cuando hay un equipo capaz de mantener ingestión, tuning y priorización de hallazgos.

Y precisamente por eso merece la pena
